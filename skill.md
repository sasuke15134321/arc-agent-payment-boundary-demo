# Skill: Agent Payment Boundary Demo for Arc Testnet

Use this skill to understand or test an AI-agent payment boundary flow on Arc Testnet.

This demo helps separate:
1. payment intent
2. approval decision
3. execution boundary
4. payment evidence
5. human review

Recommended use:
- before allowing an AI agent to execute a payment
- when reviewing payment evidence
- when designing an approval-vs-execution boundary
- when testing Arc Testnet payment boundary concepts

Do not use this as:
- an official Arc service
- an official Circle service
- a production payment gateway
- a wallet
- a settlement layer
